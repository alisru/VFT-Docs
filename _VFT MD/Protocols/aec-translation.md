# Action-Effect over Context (AE-C) Translation Method

## Two Distinct Formats — Choose Before Starting

**Passage format (Appendix A):** For sentences, verses, or blocks of text.
Structure: Original → Standard translation → Vector Duality (2–3 key terms) → Q1–7 translations.
No root analysis section. Root analysis would expand indefinitely for a multi-verse passage — the vector duality and Q-planes do the work.

**Identifier format (Appendix B):** For names, titles, or proper nouns used as functional labels.
Structure: Root Analysis → Vector Duality → Q1–7 translations applied to a key passage where the identifier appears.

Do not merge these formats. A long passage with embedded identifiers uses the Passage format — identifier decoding is woven into the vector duality and Q-plane renderings, not extracted into a separate root analysis block.

---

## Core Principle

A word is not a noun floating in a vacuum. It is a pointer to a physical or conceptual *event* taking place in reality. Its meaning is not its label — it is the action-effect geometry it describes, within the specific contextual frame in which it operates.

This is arithmetic, not interpretation. Every reading is the orthodox, dictionary-standard translation of each root word added together in sequence. No meaning is invented or retrofitted. The compound of all definitions IS the meaning.

---

## The Four Language Axioms

**Axiom I:** Words describe actions and events, not static objects.
**Axiom II:** Language operates by tracking actions within a contextual frame. The frame is not optional — it grants the word its meaning.
**Axiom III:** Words are rooted ultimately in physical reality, not arbitrary agreement. Arbitrary agreement can drift. Physical constraint cannot.
**Axiom IV:** Literacy is world-competence, not symbol decoding.

## Contextual Vector Duality

Every word or name encodes a *function*, not a fixed moral state. The same function operates in two directions:

- **Order Vector (+υ):** The function operating in alignment with the Greater Good — what the entity is *for* when correctly calibrated.
- **Chaos Vector (−υ):** The same function inverted — extraction, obstruction, or misalignment. Same mechanism, opposite direction.

The name describes the function. It does not describe the current calibration of the bearer.

## The Seven Q-Planes

| Q | Plane | Interrogative | Vocabulary |
|---|---|---|---|
| Q1 | Metaphysical | WHO | will, identity, entity, directed, allied, execute |
| Q2 | Possible | WHAT | probability, variable, data packet, potential, calculate, blueprint |
| Q3 | Physical | WHERE | physical, material, mass, kinetic, spatial, hardware, biological |
| Q4 | Lyrical | WHY | resonance, harmony, melody, soul, frequency, intrinsic meaning |
| Q5 | Logical | HOW | algorithm, protocol, calculate, system, compute, processing unit |
| Q6 | Historical | CAUSE | historical, sequence, causal, initiate, timeline, precedent |
| Q7 | Emotive | EFFECT | emotive, passion, consequence, bliss, resultant, friction |

**Critical rule on Q-plane translations:** Each Q-plane renders the **core functional claim** of the passage through that plane's vocabulary. It is NOT a word-by-word substitution of every term in the text. The Q-plane translation should be approximately the same length as the original passage — a short verse gets one or two sentences; a longer passage gets a paragraph. The plane's vocabulary is applied to the load-bearing functional terms. Most connective and structural words remain in normal prose. The output is readable flowing text in that plane's register — not a mechanical swap of every noun and verb.

## Error Prevention

**Do not** select one definition where multiple exist — sum them all.
**Do not** perform root analysis on every word of a passage — use Format A, cover 2–3 key terms in the vector duality, let the Q-planes carry the translation.
**Do not** word-substitute every term in a Q-plane translation — render the *core functional claim* in that plane's vocabulary. Readable prose, not mechanical substitution.
**Do not** confuse AE-C with Omni-Weave — AE-C works within the source language's established root definitions. Omni-Weave operates across all languages simultaneously using phonetic bigram decomposition. These are distinct tools.
